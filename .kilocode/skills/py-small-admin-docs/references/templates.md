# 文档模板索引

本文档提供了 Py Small Admin 项目中所有可用文档模板的索引和使用指南。

## 模板概述

Py Small Admin 提供三种主要文档类型的模板：

1. **指南类文档模板**（`guide-template.md`）
2. **API 文档模板**（`api-template.md`）
3. **教程类文档模板**（`tutorial-template.md`）

所有模板都位于 `assets/templates/` 目录下。

## 模板选择指南

### 根据文档类型选择

| 文档类型 | 模板文件               | 使用场景                     |
| -------- | ---------------------- | ---------------------------- |
| 指南类   | `guide-template.md`    | 安装指南、配置说明、开发指南 |
| API 文档 | `api-template.md`      | 接口说明、参数说明、示例代码 |
| 教程类   | `tutorial-template.md` | 分步教程、实战案例           |

### 根据内容选择

**使用指南类模板，如果**:

- 提供如何使用某个功能的说明
- 解释配置选项
- 描述安装步骤
- 说明开发流程

**使用 API 文档模板，如果**:

- 描述 API 接口
- 说明请求和响应格式
- 提供参数说明
- 包含示例代码

**使用教程类模板，如果**:

- 提供分步指导
- 包含实战案例
- 从零开始构建某物
- 有明确的学习目标

### 模板位置

`assets/templates/guide-template.md`

````markdown
简短的文档介绍，说明本文档的目的和适用场景。

## 前置要求

- 要求 1
- 要求 2

## 主要章节

### 子章节

内容描述...

:::tip 提示
有用的提示信息。
:::

### 代码示例

```python
# 示例代码
def example():
    pass
```
````

## 常见问题

### 问题 1

**问题**：问题描述

**答案**：问题解答

## 相关链接

- [相关文档 1](./related-doc-1.md)
- [相关文档 2](./related-doc-2.md)

````

### 使用场景

- **快速开始指南**: 帮助用户快速上手
- **安装指南**: 详细的安装步骤
- **配置指南**: 配置选项说明
- **开发指南**: 开发流程说明
- **故障排除**: 常见问题解决

### 最佳实践

1. **前置要求**: 清晰列出所有前置条件
2. **步骤清晰**: 每个步骤都有明确的说明
3. **代码示例**: 提供完整可运行的代码
4. **提示框**: 使用 `:::tip` 提供有用建议
5. **常见问题**: 预见并解答常见问题

## API 文档模板

### 模板位置

`assets/templates/api-template.md`

### 模板结构

```markdown
# API 名称

API 的简要描述。

## 请求

### 请求方法

`GET` /api/users

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id     | int  | 是   | 用户 ID |

### 请求示例

```bash
curl -X GET "http://localhost:5000/api/users?id=1"
````

## 响应

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "张三"
  }
}
```

### 响应参数

| 参数名  | 类型   | 说明     |
| ------- | ------ | -------- |
| code    | int    | 状态码   |
| message | str    | 提示信息 |
| data    | object | 数据对象 |

## 错误码

| 错误码 | 说明       |
| ------ | ---------- |
| 400    | 参数错误   |
| 404    | 资源不存在 |
| 500    | 服务器错误 |

````

### 使用场景

- **REST API 文档**: 描述 RESTful 接口
- **GraphQL API 文档**: 描述 GraphQL 查询
- **SDK 文档**: 描述 SDK 方法
- **Webhook 文档**: 描述 Webhook 事件

### 最佳实践

1. **请求方法**: 清晰标注 HTTP 方法（GET、POST、PUT、DELETE）
2. **参数表格**: 使用表格清晰列出所有参数
3. **示例代码**: 提供多种语言的示例
4. **响应示例**: 提供完整的响应示例
5. **错误码**: 列出所有可能的错误码和说明
````

````
## 模板自定义

### 修改模板

1. 复制模板文件到目标位置
2. 根据实际需求修改内容
3. 删除不需要的章节
4. 添加特定内容

### 扩展模板

基于模板添加自定义章节：

```markdown
## 自定义章节

这是自定义章节的内容。

### 子章节

更详细的内容...
````

### 组合模板

可以组合多个模板的章节：

```markdown
# 文档标题

## 前置要求（来自指南模板）

- 要求 1
- 要求 2

## API 说明（来自 API 模板）

### 请求方法

`GET` /api/example

## 实战示例（来自教程模板）

### 第一步：配置

...
```

## 模板使用流程

### 1. 选择模板

根据文档类型选择合适的模板。

### 2. 复制模板

将模板文件复制到目标位置：

```bash
cp assets/templates/guide-template.md dosc/guides/new-guide.md
```

### 3. 修改内容

根据实际需求修改模板内容：

- 更新标题和描述
- 填充实际内容
- 删除不需要的章节
- 添加特定内容

### 4. 验证格式

使用验证脚本检查文档格式：

```bash
python scripts/validate-doc.py dosc/guides/new-guide.md
```

### 5. 注册到配置

更新 `.vitepress/config.mts` 注册文档。

## 模板最佳实践

### 通用原则

1. **保持简洁**: 模板应该简洁明了
2. **易于定制**: 模板应该易于修改和扩展
3. **完整示例**: 提供完整可运行的示例
4. **清晰结构**: 保持清晰的章节结构
5. **一致风格**: 保持与项目风格一致

### 内容组织

1. **从简到繁**: 从简单到复杂
2. **先概念后实践**: 先讲解概念，再提供实践
3. **提供示例**: 始终提供示例代码
4. **使用提示框**: 使用提示框强调重要信息

### 代码示例

1. **完整性**: 代码示例应该完整可运行
2. **注释**: 添加必要的注释
3. **输出**: 标注输出结果
4. **多语言**: 提供多种语言的示例（如适用）

## 常见问题

### 如何创建新模板？

1. 复制现有模板
2. 根据需求修改
3. 保存为新模板
4. 更新本文档的索引

### 模板可以嵌套吗？

可以，模板章节可以嵌套使用：

```markdown
## 主章节

### 子章节 1（来自模板 A）

内容...

### 子章节 2（来自模板 B）

内容...
```

### 如何验证模板？

使用验证脚本检查模板：

```bash
python scripts/validate-doc.py assets/templates/guide-template.md
```

## 模板维护

### 更新模板

当发现模板需要改进时：

1. 讨论改进方案
2. 更新模板文件
3. 测试模板可用性
4. 更新本文档说明

### 版本控制

模板应该使用版本控制：

```markdown
---
title: 文档标题
description: 文档描述
template-version: 1.0
---
```

### 反馈收集

收集用户对模板的反馈：

- 哪些章节有用
- 哪些章节缺失
- 哪些内容不清楚
- 有什么改进建议
