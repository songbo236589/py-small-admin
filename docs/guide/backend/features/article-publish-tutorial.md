# 文章发布详细教程

本文档详细介绍如何使用文章发布功能将文章自动发布到第三方平台（如知乎、掘金等）。

## 功能概述

### 支持的平台

| 平台   | Cookie 验证 | 文章发布  | 备注                                        |
| ------ | ----------- | --------- | ------------------------------------------- |
| 知乎   | ✅ 已支持   | 🚧 开发中 | 使用 Cookie 认证 + 完整反检测机制           |
| 小红书 | ✅ 已支持   | ✅ 已支持 | 支持原创声明、Markdown导入、简化发布流程     |
| 掘金   | 🚧 计划中   | 🚧 计划中 | -                                           |
| CSDN   | 🚧 计划中   | 🚧 计划中 | -                                           |

## 准备工作

### 1. 安装浏览器扩展

浏览器扩展用于方便地获取各平台的登录 Cookie。

#### 安装步骤

1. **下载扩展包**
   - 从后端管理系统的"平台账号"页面下载扩展包
   - 或从项目 releases 页面下载

2. **解压扩展包**

   ```bash
   unzip py-small-admin-extension.zip -d browser-extension
   ```

3. **加载扩展**
   - 打开 `chrome://extensions/` 或 `edge://extensions/`
   - 开启"开发者模式"（右上角开关）
   - 点击"加载已解压的扩展程序"
   - 选择解压后的 `browser-extension` 目录

4. **确认安装**
   - 扩展图标会出现在浏览器工具栏
   - 可能需要固定图标以便快速访问

详见：[浏览器扩展使用指南](../../browser-extension.md)

---

### 2. 获取平台 Cookie

#### 获取知乎 Cookie

1. **登录知乎**
   - 在浏览器新标签页中打开知乎
   - 使用您的账号登录

2. **获取 Cookie**
   - 点击扩展图标
   - 找到"知乎"平台
   - 确认显示的 Cookie 数量大于 0
   - 点击"发送到后端"按钮

3. **确认结果**
   - 等待发送完成提示
   - 成功：显示"Cookie 导入成功"
   - 失败：检查错误信息

#### 获取其他平台 Cookie

目前只支持知乎平台的 Cookie 获取，其他平台（掘金、CSDN 等）正在开发中。

---

### 3. 验证账号状态

在发布文章前，建议先验证账号 Cookie 是否有效。

#### 验证步骤

1. **进入平台账号管理页面**
   - 路径：`/content/manage/platform-account`

2. **找到对应的知乎账号**
   - 在账号列表中找到要验证的账号

3. **点击"验证"按钮**
   - 点击账号操作栏的"验证"按钮

4. **等待验证完成**
   - 验证过程需要 10-30 秒
   - 请耐心等待

5. **查看验证结果**
   - 成功：账号状态更新为"有效"，显示最后验证时间
   - 失败：账号状态更新为"失效"，需要重新获取 Cookie

#### 验证限制

- 为防止频繁验证，同一账号两次验证之间必须间隔至少 5 分钟
- 验证过程会启动无头浏览器访问平台页面
- 完整的反检测机制确保验证安全

#### Windows 用户注意

在 Windows 系统上使用 Cookie 验证功能需要特殊配置：

1. 将 `.env` 文件中的 `APP_RELOAD` 设置为 `false`
2. 完全重启后端服务（使用 `python run.py`）
3. 确保使用 `run.py` 启动，而不是直接用 `uvicorn` 命令

详见：[浏览器自动化功能文档 - Windows 系统特殊配置](./browser.md#windows-系统特殊配置)

## 发布流程

### 1. 创建文章

#### 进入文章管理页面

1. 登录系统
2. 进入"内容管理"模块
3. 点击"文章管理"菜单
4. 进入文章管理页面

#### 创建新文章

1. **点击"新建文章"按钮**
   - 页面顶部的主按钮
   - 打开文章编辑抽屉

2. **填写文章信息**
   - **文章标题**：输入文章标题
   - **文章摘要**：输入文章摘要（可选）
   - **文章内容**：使用 TinyMCE 编辑器编辑文章内容
   - **选择分类**：选择文章所属分类
   - **选择标签**：选择文章标签（可选）
   - **设置状态**：选择文章状态（草稿/已发布）

3. **保存文章**
   - 点击"确定"按钮
   - 文章创建成功

#### 编辑已有文章

1. 在文章列表中找到要编辑的文章
2. 点击"编辑"按钮
3. 修改文章内容
4. 点击"确定"按钮保存

---

### 2. 发布文章

#### 单篇文章发布

1. **在文章列表中找到要发布的文章**
   - 确保文章状态为"草稿"或"已发布"

2. **点击"发布"按钮**
   - 点击文章操作栏的"发布"按钮
   - 打开发布模态框

3. **选择发布参数**
   - **选择平台**：选择要发布的平台（支持知乎、小红书）
   - **选择账号**：选择要使用的账号（必须是有效状态的账号）

4. **确认发布**
   - 点击"确定"按钮
   - 发布任务创建成功

5. **查看发布进度**
   - 系统会创建一个异步任务
   - 可以在发布管理页面查看任务进度
   - 发布完成后会更新文章状态和发布记录

#### 批量发布文章

1. **选择多篇文章**
   - 在文章列表中勾选要发布的文章
   - 可以使用筛选功能筛选出要发布的文章

2. **点击"批量发布"按钮**
   - 页面顶部的"批量发布"按钮
   - 打开发布模态框

3. **选择发布参数**
   - **选择平台**：选择要发布的平台
   - **选择账号**：选择要使用的账号

4. **确认发布**
   - 点击"确定"按钮
   - 批量发布任务创建成功
   - 每篇文章会创建一个独立的发布任务

---

### 3. 查看发布记录

#### 进入发布管理页面

1. 登录系统
2. 进入"内容管理"模块
3. 点击"发布管理"菜单
4. 进入发布管理页面

#### 查看发布记录

发布记录页面显示所有文章的发布历史。

**列表字段**：

| 字段            | 说明             |
| --------------- | ---------------- |
| **编号**        | 发布记录 ID      |
| **文章标题**    | 发布的文章标题   |
| **平台**        | 发布平台（知乎） |
| **平台文章 ID** | 平台上的文章 ID  |
| **平台链接**    | 平台文章链接     |
| **状态**        | 发布状态         |
| **错误信息**    | 失败时的错误信息 |
| **重试次数**    | 重试次数         |
| **创建时间**    | 发布任务创建时间 |

#### 发布状态

| 状态值 | 状态   | 说明     |
| ------ | ------ | -------- |
| 0      | 待发布 | 等待发布 |
| 1      | 发布中 | 正在发布 |
| 2      | 成功   | 发布成功 |
| 3      | 失败   | 发布失败 |

#### 查看发布详情

1. 点击发布记录的"查看"按钮
2. 查看详细的发布信息
3. 包括文章信息、发布参数、错误信息等

---

### 4. 重试失败发布

#### 重试步骤

1. **在发布记录中找到失败的任务**
   - 状态为"失败"的记录

2. **点击"重试"按钮**
   - 点击操作栏的"重试"按钮
   - 创建新的发布任务

3. **查看重试进度**
   - 在发布管理页面查看新的发布任务
   - 等待重试完成

## 技术实现

### 技术栈

- **浏览器自动化**：Playwright
- **Cookie 存储**：JSON 格式加密存储
- **异步任务**：Celery + RabbitMQ
- **日志记录**：完整的发布日志
- **反检测机制**：随机延迟、人类行为模拟

### 反检测配置

为降低平台封号风险，系统实现了完整的反检测机制：

#### 反检测特性

| 特性         | 默认值                    | 说明                                   |
| ------------ | ------------------------- | -------------------------------------- |
| **随机延迟** | 1-3 秒                    | 每个操作之间随机延迟，模拟人类操作节奏 |
| **页面滚动** | 1-3 次                    | 随机滚动页面，模拟浏览行为             |
| **鼠标移动** | 2-4 次                    | 随机移动鼠标位置，增加真实性           |
| **停留时间** | 成功 5-8 秒 / 失败 2-4 秒 | 根据验证结果停留不同时长               |
| **验证间隔** | 最小 5 分钟               | 防止频繁验证被检测                     |

#### 环境变量配置

在 `.env` 文件中配置反检测参数：

```bash
# 反检测配置（降低平台封号风险）
CONTENT_HUMAN_BEHAVIOR_ENABLED=true        # 是否启用人类行为模拟
CONTENT_RANDOM_DELAY_MIN=1.0               # 操作间最小随机延迟时间（秒）
CONTENT_RANDOM_DELAY_MAX=3.0               # 操作间最大随机延迟时间（秒）
CONTENT_VERIFY_INTERVAL_MIN=300            # 最小验证间隔（秒），默认 5 分钟
CONTENT_STAY_TIME_SUCCESS_MIN=5.0          # 验证成功后最小停留时间（秒）
CONTENT_STAY_TIME_SUCCESS_MAX=8.0          # 验证成功后最大停留时间（秒）
CONTENT_STAY_TIME_FAILED_MIN=2.0           # 验证失败后最小停留时间（秒）
CONTENT_STAY_TIME_FAILED_MAX=4.0           # 验证失败后最大停留时间（秒）
CONTENT_SCROLL_COUNT_MIN=1                 # 最小滚动次数
CONTENT_SCROLL_COUNT_MAX=3                 # 最大滚动次数
CONTENT_MOUSE_MOVE_COUNT_MIN=2             # 最小鼠标移动次数
CONTENT_MOUSE_MOVE_COUNT_MAX=4             # 最大鼠标移动次数
```

#### 人类行为模拟

发布过程会模拟真实用户的浏览行为：

1. **随机延迟**：每个操作之间随机延迟 1-3 秒
2. **页面滚动**：随机滚动 1-3 次，每次滚动 100-500 像素
3. **鼠标移动**：随机移动鼠标 2-4 次，模拟用户浏览
4. **停留时间**：根据发布结果停留不同时长（成功 5-8 秒，失败 2-4 秒）

这些行为会自动执行，无需额外配置。如需禁用，将 `CONTENT_HUMAN_BEHAVIOR_ENABLED` 设置为 `false`。

## 常见问题

### 1. 发布失败

**症状**：发布状态显示"失败"，有错误信息

**常见错误及解决方案**：

#### Cookie 已失效

**错误信息**：`Cookie 已失效，请重新获取`

**解决方案**：

1. 重新登录知乎
2. 使用浏览器扩展重新获取 Cookie
3. 验证账号状态
4. 重新发布文章

#### 平台页面结构变化

**错误信息**：`无法找到发布按钮`

**解决方案**：

1. 检查知乎页面是否有更新
2. 联系开发者更新发布器
3. 暂时使用手动发布

#### 网络超时

**错误信息**：`网络超时`

**解决方案**：

1. 检查网络连接
2. 尝试增加超时时间
3. 检查防火墙设置

#### 内容格式错误

**错误信息**：`文章内容格式不正确`

**解决方案**：

1. 检查文章内容是否为 Markdown 格式
2. 检查是否有特殊字符
3. 简化文章内容后重试

---

### 2. 发布超时

**症状**：发布状态一直显示"发布中"，超过 5 分钟

**可能原因**：

1. **网络速度慢**
   - 检查网络连接
   - 尝试使用代理

2. **平台响应慢**
   - 这是正常现象，请耐心等待
   - 可以增加超时时间

3. **浏览器卡死**
   - 重启后端服务
   - 检查系统资源

---

### 3. 批量发布部分失败

**症状**：批量发布 10 篇文章，只有 8 篇成功

**可能原因**：

1. **部分文章内容有问题**
   - 检查失败的文章
   - 修改内容后重试

2. **账号 Cookie 失效**
   - 验证账号状态
   - 重新获取 Cookie

3. **平台限流**
   - 减少批量发布的数量
   - 增加发布间隔

---

### 4. 无法看到发布按钮

**症状**：文章列表中没有"发布"按钮

**可能原因**：

1. **没有有效的平台账号**
   - 先添加有效的平台账号
   - 确保账号状态为"有效"

2. **权限不足**
   - 确认用户有发布权限
   - 联系管理员检查权限配置

3. **文章状态不正确**
   - 确保文章状态为"草稿"或"已发布"
   - 草稿稿状态的文章也可以发布

---

### 5. Windows 系统发布失败

**症状**：发布时提示"验证过程出错: NotImplementedError"

**原因**：Windows 系统的默认事件循环不支持 Playwright 创建子进程

**解决方案**：

1. 将 `.env` 文件中的 `APP_RELOAD` 设置为 `false`
2. 完全重启后端服务（使用 `python run.py`）
3. 确保使用 `run.py` 启动，而不是直接用 `uvicorn` 命令

详见：[浏览器自动化功能文档 - Windows 系统特殊配置](./browser.md#windows-系统特殊配置)

## 最佳实践

### 1. 定期验证账号

- 建议每周验证一次账号状态
- 及时更新过期的 Cookie
- 避免使用失效的 Cookie 发布

### 2. 合理设置发布频率

- 避免短时间内大量发布
- 建议每次发布间隔至少 5 分钟
- 避免被平台限流

### 3. 检查发布结果

- 发布后及时查看发布记录
- 确认文章是否成功发布
- 失败的文章及时重试

### 4. 优化文章内容

- 确保文章内容质量
- 使用 Markdown 格式
- 避免敏感词汇

### 5. 备份重要文章

- 发布前备份文章内容
- 保留原始内容
- 方便重新发布
