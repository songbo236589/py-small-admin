# 话题管理功能

本文档介绍如何使用话题管理功能从知乎平台抓取热门话题，并使用话题创建文章。

## 功能概述

### 功能介绍

话题管理功能允许用户从知乎平台抓取推荐问题/话题，帮助快速找到写作灵感，辅助 AI 写作功能。

### 核心功能

| 功能         | 描述                                       |
| ------------ | ------------------------------------------ |
| **话题抓取** | 从知乎推荐问题页面抓取热门话题（手动触发） |
| **话题存储** | 将抓取的话题存储到数据库，支持去重         |
| **话题展示** | 在管理后台展示已抓取的话题列表             |
| **话题使用** | 选择话题后自动填充到文章编辑表单           |
| **状态管理** | 支持话题的收藏、使用状态标记               |

### 技术特点

- **手动触发**：用户主动点击按钮抓取，按需使用
- **同步处理**：抓取过程用户可见，有 Loading 状态提示
- **去重机制**：自动过滤已存在的话题，支持数据更新
- **热度计算**：根据浏览量和回答数计算话题热度

### 使用场景

- 寻找写作灵感
- 获取热门话题
- 辅助 AI 内容生成
- 跟踪平台热点

## 使用方法

### 1. 获取知乎账号 Cookie

#### 使用浏览器扩展获取（推荐）

1. **安装浏览器扩展**
   - 在平台账号管理页面下载扩展包
   - 解压并加载到浏览器（开发者模式）
   - 详见：[浏览器扩展使用指南](../../browser-extension.md)

2. **获取 Cookie**
   - 在浏览器中登录知乎
   - 点击扩展图标
   - 点击"发送到后端"按钮

#### 手动添加 Cookie

1. 进入平台账号管理页面
2. 点击"添加账号"按钮
3. 选择平台"知乎"
4. 填写账号名称
5. 手动粘贴 Cookie 数据（JSON 格式）
6. 点击"确定"提交

**Cookie 格式示例**：

```json
[
  {
    "name": "z_c0",
    "value": "xxx",
    "domain": ".zhihu.com",
    "path": "/",
    "secure": true,
    "httpOnly": true,
    "expirationDate": 1735660800
  }
]
```

---

### 2. 验证账号状态

在抓取话题前，建议先验证账号 Cookie 是否有效。

#### 验证步骤

1. 进入平台账号管理页面
2. 找到对应的知乎账号
3. 点击"验证"按钮
4. 等待验证完成（约 10-30 秒）

#### 验证结果

| 状态     | 说明                  | 操作                |
| -------- | --------------------- | ------------------- |
| **有效** | Cookie 有效，可以使用 | 可以开始抓取        |
| **失效** | Cookie 已失效         | 需要重新获取 Cookie |
| **过期** | Cookie 即将过期       | 建议重新获取 Cookie |

#### Windows 用户注意

在 Windows 系统上使用 Cookie 验证功能需要特殊配置：

1. 将 `.env` 文件中的 `APP_RELOAD` 设置为 `false`
2. 完全重启后端服务（使用 `python run.py`）
3. 确保使用 `run.py` 启动，而不是直接用 `uvicorn` 命令

详见：[浏览器自动化功能文档 - Windows 系统特殊配置](./browser.md#windows-系统特殊配置)

---

### 3. 抓取话题

#### 抓取步骤

1. **进入话题管理页面**
   - 路径：`/content/manage/topic`

2. **点击"抓取话题"按钮**
   - 页面顶部的主按钮
   - 打开抓取模态框

3. **选择抓取参数**
   - 选择知乎账号（状态为有效的账号）
   - 选择抓取数量（10/20/50个）
   - 点击"开始抓取"按钮

4. **等待抓取完成**
   - 按钮显示 Loading 状态
   - 预计耗时 10-30 秒
   - 等待抓取完成提示

5. **查看抓取结果**
   - 系统自动抓取并存储话题到数据库
   - 显示成功提示：新增 X 个，更新 Y 个

#### 抓取参数说明

| 参数         | 说明                                       |
| ------------ | ------------------------------------------ |
| **知乎账号** | 选择要使用的知乎账号，必须是有效状态的账号 |
| **抓取数量** | 选择要抓取的话题数量，可选 10/20/50 个     |

#### 抓取过程

```
┌─────────────────────────────────────────────────────────────┐
│                    抓取过程                                      │
└─────────────────────────────────────────────────────────────┘

1. 验证账号状态
   │
   ▼
2. 启动 Playwright 浏览器（Chromium）
   │
   ▼
3. 创建浏览器上下文（设置 User-Agent、Viewport）
   │
   ▼
4. 访问知乎推荐问题页面
   │
   ▼
5. 添加 Cookies 到浏览器上下文
   │
   ▼
6. 等待页面加载完成
   │
   ▼
7. 模拟人类行为（随机滚动、鼠标移动）
   │
   ▼
8. 解析话题数据（标题、描述、浏览量、回答数等）
   │
   ▼
9. 存储到数据库（去重）
   │
   ▼
10. 关闭浏览器，返回结果
```

### 4. 查看话题列表

#### 话题列表页面

进入话题管理页面后，可以看到已抓取的话题列表。

#### 列表字段

| 字段         | 说明                       | 操作 |
| ------------ | -------------------------- | ---- |
| **编号**     | 话题 ID                    | -    |
| **问题标题** | 问题标题，点击可跳转到知乎 | -    |
| **平台**     | 平台标识（知乎）           | -    |
| **分类**     | 话题分类                   | 筛选 |
| **状态**     | 未使用/已使用/已收藏       | 筛选 |
| **热度**     | 热度分数                   | 排序 |
| **浏览量**   | 问题浏览量                 | 排序 |
| **回答数**   | 问题回答数                 | 排序 |
| **抓取时间** | 话题抓取时间               | 排序 |
| **操作**     | 使用、收藏、删除           | -    |

#### 筛选和排序

- **筛选**：按平台、状态、分类筛选
- **排序**：按热度、浏览量、回答数、抓取时间排序
- **搜索**：按标题关键词搜索

### 5. 使用话题创建文章

#### 使用步骤

1. **选择感兴趣的话题**
   - 在话题列表中找到感兴趣的话题
   - 查看话题的标题、描述、浏览量等信息

2. **点击"使用"按钮**
   - 点击话题操作栏的"使用"按钮
   - 系统自动标记话题为"已使用"

3. **编辑文章内容**
   - 系统自动打开文章编辑抽屉
   - 自动填充标题和描述
   - 编辑文章内容

4. **保存文章**
   - 点击"确定"保存文章
   - 文章创建成功

#### 自动填充内容

使用话题创建文章时，系统会自动填充以下内容：

| 字段         | 自动填充内容                            |
| ------------ | --------------------------------------- |
| **文章标题** | 话题标题                                |
| **文章摘要** | 话题描述                                |
| **文章内容** | Markdown 格式，包含标题、描述和参考链接 |

**自动填充示例**：

```markdown
# 如何学习 Python 编程？

我是编程初学者，想系统地学习 Python，请提供学习路线和建议。

> 参考链接：https://www.zhihu.com/question/12345678
```

### 6. 话题管理

#### 收藏话题

1. 在话题列表中找到要收藏的话题
2. 点击"收藏"按钮（星星图标）
3. 话题状态变为"已收藏"

#### 取消收藏

1. 再次点击"收藏"按钮
2. 话题状态恢复为"未使用"

#### 查看已收藏话题

- 在状态筛选中选择"已收藏"
- 只显示已收藏的话题

---

#### 删除话题

1. 在话题列表中找到要删除的话题
2. 点击"删除"按钮
3. 确认删除操作
4. 话题从数据库中删除

#### 批量删除

1. 勾选要删除的话题
2. 点击"批量删除"按钮
3. 确认删除操作
4. 话题批量删除

## 技术实现

### 抓取流程

```
┌─────────────────────────────────────────────────────────────┐
│                    话题抓取流程                                 │
└─────────────────────────────────────────────────────────────┘

┌─────────┐    ┌────────────┐    ┌──────────────┐    ┌─────────┐
│  1.导航  │───▶│ 2.创建临时  │───▶│ 3.上传Markdown│───▶│ 4.发布  │
│ 到编辑器 │    │    文件     │    │     文件      │    │  文章   │
└─────────┘    └────────────┘    └──────────────┘    └─────────┘
                   │                   │                    │
                   ▼                   ▼                    ▼
            ┌────────────┐    ┌──────────────┐    ┌─────────────┐
            │ 添加版权声明│    │ 点击更多按钮 │    │ 两步确认发布 │
            │ (文末追加) │    │ 选择导入文档 │    │ (设置→发布) │
            └────────────┘    └──────────────┘    └─────────────┘
```

### 反检测机制

为降低平台封号风险，系统实现了完整的反检测机制：

| 特性         | 默认值                    | 说明                                   |
| ------------ | ------------------------- | -------------------------------------- |
| **随机延迟** | 1-3 秒                    | 每个操作之间随机延迟，模拟人类操作节奏 |
| **页面滚动** | 1-3 次                    | 随机滚动页面，模拟浏览行为             |
| **鼠标移动** | 2-4 次                    | 随机移动鼠标位置，增加真实性           |
| **停留时间** | 成功 5-8 秒 / 失败 2-4 秒 | 根据验证结果停留不同时长               |
| **验证间隔** | 最小 5 分钟               | 防止频繁验证被检测                     |

### 数据模型

#### 话题表 (content_topics)

| 字段                 | 类型         | 说明                               |
| -------------------- | ------------ | ---------------------------------- |
| id                   | INT          | 主键 ID                            |
| platform             | VARCHAR(20)  | 平台标识（zhihu）                  |
| platform_question_id | VARCHAR(50)  | 平台问题 ID                        |
| title                | VARCHAR(500) | 问题标题                           |
| description          | TEXT         | 问题描述                           |
| url                  | VARCHAR(500) | 问题链接                           |
| view_count           | INT          | 浏览量                             |
| answer_count         | INT          | 回答数                             |
| follower_count       | INT          | 关注者数                           |
| category             | VARCHAR(100) | 分类/话题标签                      |
| hot_score            | INT          | 热度分数                           |
| author_name          | VARCHAR(100) | 提问者昵称                         |
| author_url           | VARCHAR(200) | 提问者主页                         |
| status               | TINYINT      | 状态：0=未使用, 1=已使用, 2=已收藏 |
| fetched_at           | DATETIME     | 抓取时间                           |
| created_at           | DATETIME     | 创建时间                           |
| updated_at           | DATETIME     | 更新时间                           |

### 热度计算

热度分数根据浏览量和回答数计算：

```python
hot_score = view_count + answer_count * 10
```

## 常见问题

### 1. 抓取失败

**症状**：点击"开始抓取"后提示"抓取失败"

**排查步骤**：

1. **检查账号状态**
   - 确认账号状态为"有效"
   - 如果状态为"失效"，需要重新获取 Cookie

2. **检查 Cookie**
   - 确认 Cookie 未过期
   - 尝试重新获取 Cookie

3. **检查网络连接**
   - 确保可以访问知乎网站
   - 检查防火墙设置

4. **检查 Playwright**
   - 确认 Playwright 已安装：`playwright --version`
   - 确认浏览器已安装：`playwright install chromium`

---

### 2. Cookie 过期

**症状**：验证账号时提示"Cookie 已失效"

**解决方案**：

1. **重新登录知乎**
   - 在浏览器中重新登录知乎
   - 使用浏览器扩展重新获取 Cookie

2. **更新账号 Cookie**
   - 进入平台账号管理页面
   - 点击"编辑"按钮
   - 更新 Cookie 数据
   - 保存并重新验证

---

### 3. 抓取速度慢

**症状**：抓取过程耗时超过 30 秒

**可能原因**：

1. **网络速度慢**
   - 检查网络连接
   - 尝试使用代理

2. **知乎页面加载慢**
   - 这是正常现象，请耐心等待

3. **浏览器性能问题**
   - 尝试减少抓取数量

---

### 4. 抓取到重复话题

**症状**：抓取结果显示"更新 X 个"，没有新增话题

**说明**：

- 这是正常现象，系统会自动去重
- 已存在的话题会更新数据（浏览量、回答数等）
- 可以查看话题的更新时间确认数据是否更新

---

### 5. Windows 系统验证失败

**症状**：验证账号时提示"验证过程出错: NotImplementedError"

**原因**：Windows 系统的默认事件循环不支持 Playwright 创建子进程

**解决方案**：

1. 将 `.env` 文件中的 `APP_RELOAD` 设置为 `false`
2. 完全重启后端服务（使用 `python run.py`）
3. 确保使用 `run.py` 启动，而不是直接用 `uvicorn` 命令

详见：[浏览器自动化功能文档 - Windows 系统特殊配置](./browser.md#windows-系统特殊配置)

---

### 6. 无法看到抓取按钮

**症状**：话题管理页面没有"抓取话题"按钮

**可能原因**：

1. **没有有效的知乎账号**
   - 先添加有效的知乎账号
   - 确保账号状态为"有效"

2. **权限不足**
   - 确认用户有话题管理权限
   - 联系管理员检查权限配置

## 最佳实践

### 1. 定期抓取话题

- 建议每天或每周抓取一次
- 关注热门话题变化
- 及时获取新的写作灵感

### 2. 合理设置抓取数量

- 初次使用建议抓取 10 个
- 根据需要调整抓取数量
- 避免一次抓取过多

### 3. 及时使用话题

- 话题热度会随时间变化
- 建议及时使用热门话题
- 定期清理过时话题

### 4. 使用收藏功能

- 收藏感兴趣的话题
- 方便后续查找和使用
- 建立个人话题库
